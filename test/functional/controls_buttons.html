Button Tests

Using the BUTTON element

  • First button is normal. Should have hover state, focus state, and down state.
  • Second button is disabled. Should not change state when hovered over, clicked on, or focused.
  • Third button is "primary" — denotes most common action. Should look different from the first button. May have distinct hover and down states, if necessary.
An anchor

A test to ensure that buttons made from different HTML elements are styled to look as similar as possible.

A basic S2.UI.ButtonSet with two buttons.

  • The buttons should appear grouped. It's left to an individual theme to decide how to convey the grouping. In the default theme, the buttons appear "joined" at the spot where they meet; this spot should not have any rounded corners or empty space.

A ButtonSet with two radio buttons.

  • The buttons should behave like radio buttons. That means:

    • There should not be more than one in the "active" state at once.
    • An active button should not change state when it is clicked on.
